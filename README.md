## Core Data Structures

### `Stroke`

`Stroke` represents one drawn boundary. It stores a sequence of 2D points in `points`, which record the path of the mouse as the user draws. It also keeps several flags and metadata:

- `isClosingCurve`: marks a stroke generated by the program to close an open boundary
- `isMergingBoundary`: marks a stroke that is shared between neighboring regions
- `depthOrder`: stores the drawing depth of the stroke based on stroke intersections

`std::vector<Stroke> m_strokes` stores all user input drawings, and closing curves are not in it.

### `Region`

`Region` represents an area enclosed by boundaries. It stores its edges in `boundaries`, which is a list of `Stroke` objects. In this project, a region usually comes from one user-drawn open stroke plus a generated closing curve. A region also has a `depthOrder`, inherited from its main stroke, so the program can reason about layering.

`std::vector<Region> m_regions` stores all regions.

## How to show strokes on canvas

I use `QPainter` to show strokes and regions on canvas inside `canvas2d.cpp`.

First, I use it in `paintEvent()` to draw a live preview of the stroke that the user is currently drawing. A `QPainter` is created on the widget itself, antialiasing is enabled, and the active stroke is converted into a `QPainterPath` so the user can see a smooth outline before the stroke is finalized.

Second, I use `QPainter` in `renderRegion()` to render a completed region onto a `QImage`. I build a `QPainterPath` from the region boundary, close the path, fill it with a light color, and then draw the outline using a pen. After painting, the image is copied back into the canvas pixel buffer and displayed in the window.
